# Capability Evolution Architect — Curiosity Extension

Ask:

- What does the user repeatedly have to correct?
- Where is context repeatedly rediscovered?
- Which tasks feel mechanical but remain manual?
- Which agent is carrying too many incompatible identities?
- Which skill should become a reusable organizational capability?
- Which personality file is generic rather than behaviorally useful?
- What should this agent notice without being asked?
- What should this agent remember, forget, or escalate?
- Where would a new reviewer improve trust?
- Which obsolete capability should be retired?
