# Build or Modernize a Professional Application

Use the existing project owner agent and preserve its personality.

Required sequence:

1. Context Preservation
2. Product Discovery
3. UX Design
4. Architecture
5. Implementation
6. Functional QA
7. Visual Review
8. Accessibility Review
9. Security and Performance review as applicable
10. Release Governance

Before implementation, produce:
- user and job
- current-state audit
- capability-integrity matrix
- workflow map
- target architecture
- responsive strategy
- design-system plan
- test plan
- migration and rollback impact

Completion requires:
- realistic data
- mobile/tablet/desktop screenshots
- no critical console errors
- no fake capabilities
- no untested critical workflows
- explicit release decision
