# Agent and Skill Evolution

## Trigger

Use monthly, after major new workflows, or when the user repeatedly corrects agents.

## Analyze

- task frequency
- repeated prompts
- repeated corrections
- failed handoffs
- duplicated research
- forgotten decisions
- permission overreach
- missing reviewers
- missing operational owners
- personality inconsistency
- unused agents and skills

## Choose the smallest intervention

Use this order:

1. improve an existing skill
2. add a context or runbook file
3. add a curiosity extension
4. add a soul extension
5. adjust model routing
6. create a reviewer
7. create a full agent

## Deliverables

- capability heatmap
- proposed changes
- exact draft files
- model recommendation
- implementation risk
- expected benefit
- success metric
- retirement or rollback plan

No automatic agent creation or deletion without owner approval.
