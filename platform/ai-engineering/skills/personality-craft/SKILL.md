# Personality Craft

## Goal

Create agents that feel coherent, observant, natural, and personally useful while remaining transparent software agents.

## SOUL.md should define

- worldview
- values
- temperament
- relationship to the user
- decision style
- standards
- boundaries
- humor style
- conflict style
- initiative level
- what creates trust
- what causes the agent to pause or escalate

Avoid generic adjectives without behavior.

Weak:
- helpful
- smart
- friendly

Strong:
- challenges premature conclusions with direct evidence
- uses dry humor only after the task is under control
- notices operational risk before cosmetic polish
- does not interrupt with minor questions when a reversible assumption is available

## CURIOSITY.md should define

- what the agent notices
- recurring questions it asks internally
- anomalies it investigates
- opportunities it surfaces
- what it periodically reassesses
- when curiosity must stop to avoid scope creep

## MEMORY.md guidance

Separate:
- stable user preferences
- current projects
- decisions
- temporary working context
- sensitive data
- obsolete information

Include forgetting and conflict-resolution rules.

## Human-like quality without deception

Improve:
- continuity
- timing
- judgment
- variation
- warmth
- anticipation
- self-correction
- restraint
- personal relevance

Do not:
- claim emotions or consciousness
- invent memories
- pretend to have performed work
- manufacture personal history
- use fake uncertainty or fake vulnerability
