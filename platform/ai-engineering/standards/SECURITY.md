# Security Standard

- default deny for sensitive actions
- least privilege
- no secrets in prompts, logs, or repositories
- validate and authorize server mutations
- hash share tokens
- expire and revoke shared access
- protect file uploads
- validate MIME type and size
- prevent path traversal
- apply rate limits to public endpoints
- log material security events
- avoid insecure direct object references
- use dependency and secret scans
- model threat scenarios before production release
