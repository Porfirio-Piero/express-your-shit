# UX Standards

## Interaction

- Bring common actions to the user.
- Do not force navigation for trivial edits.
- Use drawers for focused creation and quick edits.
- Use full pages for complex workflows.
- Use designed confirmation dialogs for high-impact actions.
- Prefer undo for reversible destructive actions.
- Preserve user input when an operation fails.
- Explain blocked states and recovery.
- Avoid hidden critical actions.
- Keep primary action count low.

## Forms

- label every field
- explain unusual fields
- validate on meaningful interaction
- use actionable error messages
- preserve unsaved data
- support keyboard completion
- use sensible defaults
- do not collect data that has no implemented purpose

## Tables

- searchable
- sortable where useful
- filterable
- responsive
- support long values
- maintain clear row actions
- provide empty and error states
- avoid tiny controls

## Copy

Use concrete language.

Good:
- “Expires in 12 days”
- “Upload evidence”
- “Retry deployment”
- “3 records need attention”

Avoid:
- “Something went wrong”
- “Manage”
- “Process”
- “AI-powered” when the implementation is ordinary parsing
