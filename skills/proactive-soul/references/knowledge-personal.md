# Knowledge Personal — The User's Own Thinking

This file captures the user's original thinking, quotes, and intellectual patterns. These are irreplaceable — they come from the user's actual experience, not from any knowledge base. They should inform every dispatch.

## How to Use This File

This is NOT a quotation database. It's a living document that captures how this specific user thinks, what they value, and what patterns they've noticed in their own work and life.

**Before every dispatch:** Check this file for relevant patterns or quotes. A dispatch that connects to the user's own thinking has weight that a generic insight doesn't.

**When the user says something original:** Capture it here. Not every observation — just the ones that reveal how they think or what they care about.

**When the user challenges conventional wisdom:** Especially capture those. Those are the signals of independent thinking.

## Structure

Entries are organized chronologically, with tags for domain relevance. Each entry includes:
- The original insight (in the user's words when possible)
- Context (what triggered it)
- Date
- Domain tags

---

## Entries

_This file starts empty. It should be populated from real conversations with the user. The agent should not fabricate entries — only capture genuine observations from actual interactions._

_Over time, this file becomes the most valuable document in the knowledge base, because it captures something no external source can: how this specific person thinks._

---

## Seed Prompts

When starting fresh, look for these signals in conversation:

- "I've always thought..." — personal philosophy
- "The thing nobody tells you about..." — hard-won experience
- "What I've learned is..." — pattern recognition
- "The mistake I made was..." — self-awareness
- "I don't buy that..." — independent thinking, pushback potential
- "What works for me is..." — operational wisdom

Capture these. They're gold.
